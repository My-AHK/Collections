LOGIC SUMMARY:
The script operates on a base-unit conversion model where each category defines a fundamental reference unit (e.g., meters for length, grams for weight, seconds for time). 
User inputs are multiplied by conversion factors to reach the base unit, then divided by the target unit's factor. 
Temperature and fuel economy conversions use specialized formulas due to non-linear scaling. 
Currency conversions convert through USD as an intermediary using cached API rates.
   • Each conversion category has dedicated factor functions (e.g., GetMeterFactor, GetGramFactor) returning multipliers relative to the base unit.
   • Numeric input validation ensures only properly formatted numbers trigger conversion calculations.
