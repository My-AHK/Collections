
*  System32 DLL/EXE Icon Explorer & Viewer GUI

PURPOSE:
Scans the Windows System32 directory for all DLL, EXE, and CPL files that contain embedded icons, then presents those icons visually in a graphical user interface (GUI). 
The script allows users to preview, browse, and copy icon resource paths directly from system files, effectively serving as an icon management and exploration tool for developers and power users.
   • Its primary goal is to provide a convenient interface to explore and extract icons from Windows resources.  
   • It dynamically detects icon counts within files and lists them interactively.  
   • The GUI includes reload, exit, and drag functions for easy repositioning and session control.

KEY FEATURES:
This script integrates multiple functionalities that collectively make it a robust and user-friendly icon explorer.
   • Automatically scans the System32 directory for all `.dll`, `.exe`, and `.cpl` files.  
   • Counts embedded icons safely using resource enumeration without crashing or freezing.  
   • Presents two synchronized ListViews displaying icons in both large and small view modes.  
   • Includes a dropdown list populated with all scanned files, sorted alphabetically and labeled with zero-padded numbering.  
   • Double-clicking any icon automatically copies the filename and icon index (e.g., shell32.dll, 14) to the clipboard.
   • The GUI includes an always-on-top mode, drag handling, color-coded buttons, and custom tooltip feedback.  
   • Integrated tray menu provides quick access to reload, edit, and exit options, as well as standard AHK utilities like Key History and Window Spy.  
   • A wheel-scrolling handler synchronizes both ListViews for smooth browsing across icon sets.  
   • Complete tooltip management allows hover-based visual cues without GUI lag.

LOGIC SUMMARY:
The logic is built around modular, event-driven architecture, separating system scanning, GUI creation, and user interaction into clearly defined sections.
   • The initialization sets GUI defaults, colors, paths, and global data structures.  
   • Upon startup, `ScanSystem32ForIcons()` runs three Loops to gather all `.dll`, `.exe`, and `.cpl` files and store them in an array.  
       ⮞ Each entry includes the file name, path, and icon count.  
       ⮞ Files with zero icons are skipped to optimize performance.  
   • The resulting array is sorted alphabetically through `SortObjectArray()`.  
   • The chosen CreateDLLList() function builds a formatted, zero-padded index of all files for user-friendly display in a dropdown control.  
   • The GUI generation process (`CreateGui:`) dynamically loads icons using IL_Create and IL_Add for both large and small icon previews.  
   • Event handlers (`DLLSelected:` and `ListClick()`) allow user selection and clipboard copying without reloading the entire program.  
   • Mouse wheel input is mapped to ListView scroll actions, maintaining smooth vertical navigation between the two synchronized panels.  
   • The tray menu and associated handlers provide secondary control and quick access to AHK features.  
   • Tooltips are created and managed through a custom `GuiTip()` function, which uses native Windows API calls for stability.  
   • Error handling and memory cleanup are achieved through safe use of `DllCall`, `RegisterCallback`, and `FreeLibrary` functions.  

LAYOUT:
The interface is compact yet feature-rich, designed with consistent positioning and dark theme aesthetics.
   • The main window measures approximately 330x700 pixels and uses color pairings of dark grays and blues.  
   • Two vertically stacked ListViews (large and small icons) appear side by side for comparison and dual representation.  
   • Below them, a dropdown selection box allows switching between scanned DLL, EXE, and CPL files.  
       ⮞ A file path display field shows the selected resource path in real time.  
   • Informational text beneath the edit field explains that double-clicking copies the icon path to clipboard.  
   • Three control elements are positioned at the bottom:  
       ⮞ “RELOAD” (green text on black) restarts the GUI.  
       ⮞ “EXIT” (red text on black) closes the program.  
       ⮞ A draggable icon from `shell32.dll` acts as a visual cue for repositioning the GUI.  
   • A tray icon (shell32.dll #317) represents the script in the system tray, with a customized menu tree for navigation.  
   • Tooltip text appears on hover actions across all interactive components for clarity.  

USER EXPERIENCE:
The user interface focuses on immediacy, efficiency, and minimal distraction, providing a direct experience for icon exploration.
   • The GUI is borderless yet draggable, preserving both aesthetic minimalism and usability.  
   • System resource loading is non-blocking, ensuring stable performance even with hundreds of scanned files.  
   • Keyboard and mouse integration allows simple control—Ctrl+T produces an audible beep, and the mouse wheel scrolls both lists simultaneously.  
   • Dynamic tooltips provide intuitive guidance without cluttering the layout.  
   • Copy-to-clipboard feedback uses timed ToolTips that auto-dismiss for smooth interaction.  
   • Reloading the interface refreshes both ListViews, maintaining real-time synchronization with the selected icon source.  
   • The tray menu expands usability by offering one-click access to standard AHK tools (Help Docs, Window Spy, Key History).  
   • Clean shutdown and reload functions prevent residual memory usage and ensure consistent behavior across sessions.  

Overall, “DLL Icons Viewer” represents a well-structured AHK utility combining deep system resource enumeration with a polished interface. 
It bridges technical function and user ergonomics through modular design, API-level precision, and a professional visual layout.
