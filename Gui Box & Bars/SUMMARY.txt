
*  Custom GUI box, bar, and angled line renderer


PURPOSE:
This script demonstrates how to visually construct custom graphical elements within an AutoHotkey GUI using simple controls. 
It serves as a reusable toolkit for drawing boxes, straight bars, and angled lines without relying on external libraries, making it ideal for lightweight UI decoration or visual indicators.
   • Provides a clean method for drawing shapes using built-in GUI controls.
   • Focuses on flexibility through parameterized drawing functions.
   • Acts as both a demo and a reusable utility library.

KEY FEATURES:
The script includes multiple drawing routines that allow for creating outlined boxes, solid horizontal/vertical bars, and diagonal lines at various angles. 
Each feature is demonstrated within the GUI using labeled examples for clarity.
   • 4-color box outline with independent edge colors.
   • Solid bar rendering for both horizontal and vertical orientations.
   • Diagonal line drawing with adjustable angle (0–90 degrees).
       ⮞ Supports both upward (/) and downward (\) directions.
   • Variable thickness support for all shapes.
   • Combined bar layering for visual effects.
   • Click-and-drag GUI movement via message handling.

LOGIC SUMMARY:
The script builds all visual elements using the Progress control as a drawing primitive. 
Each function calculates positions and dimensions, then places colored rectangles to simulate lines or borders. 
Diagonal lines are constructed pixel-by-pixel using trigonometric calculations.
   • boxline() calculates edge positions and draws four borders.
   • barLine() creates a filled rectangle for straight bars.
   • barLineAngle() uses Tan(angle) to determine pixel offsets.
       ⮞ Converts degrees to radians for accurate slope calculation.
       ⮞ Iteratively plots small segments to form a diagonal line.
   • Edge cases (0° and 90°) are optimized as straight bars.

LAYOUT:
The GUI is structured as a visual showcase, presenting labeled examples of each drawing capability. 
Elements are spaced across the window to demonstrate different orientations, angles, and thicknesses.
   • Top section displays a multi-colored bordered box.
   • Middle sections show horizontal and vertical bars.
   • Right side demonstrates diagonal lines at 30°, 45°, and 60°.
   • Bottom section compares thickness variations and layered bars.
   • Always-on-top, borderless tool window with dark theme.

USER EXPERIENCE:
The interface is simple and interactive, allowing users to visually inspect how each function behaves. 
The draggable window and labeled examples make it easy to understand and adapt the routines for custom use.
   • Clean, minimal GUI with clear labeling.
   • Immediate visual feedback for all drawing functions.
   • Drag-to-move functionality enhances usability.
   • No dependencies or complex setup required.
