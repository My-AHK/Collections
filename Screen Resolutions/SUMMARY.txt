
*  Displays detailed monitor info and display modes.

PURPOSE:
This script retrieves detailed information about all active monitors connected to the system by combining data from both Windows Management Instrumentation (WMI) and AutoHotkey's SysGet command.
It gathers monitor identification, physical dimensions, supported display modes, and refresh rates, then presents the information in a color-coded GUI while also copying the complete report to the clipboard.
    • Connects to WMI for monitor hardware information.
    • Uses SysGet to identify monitor names and the primary display.
    • Displays monitor details in an easy-to-read format.
    • Automatically copies the generated report to the clipboard.

KEY FEATURES:
The script combines information from multiple Windows sources to provide a more complete monitor report than either source alone.
It also includes error handling, monitor size calculations, refresh rate calculations, clipboard support, and a multi-column graphical display.
    • Detects the primary monitor.
    • Enumerates all active monitors.
    • Retrieves monitor instance names.
    • Retrieves monitor names reported by SysGet.
    • Calculates physical monitor size in inches from millimeter measurements.
    • Lists supported resolutions and refresh rates.
    • Limits displayed display modes to the first 50 entries per monitor.
    • Uses different text colors for each monitor section.
    • Displays the GUI as always-on-top.
    • Allows the report to be copied without manually selecting text.

LOGIC SUMMARY:
The script begins by establishing a WMI connection and verifies that monitor information can be queried successfully.
It then determines the number of connected monitors, records their names, and queries each active monitor for its supported display modes.
For every monitor found, the script identifies whether it is the primary display, determines the largest supported resolution to estimate the monitor's physical dimensions, calculates its diagonal screen size, and formats all collected information into a readable report.
Finally, it combines every monitor report into a single clipboard string before constructing a graphical interface that displays each monitor's information in separate colored columns.
    • Connects to the WMI monitor provider.
    • Detects the primary monitor number.
    • Retrieves the total monitor count.
    • Stores monitor names returned by SysGet.
    • Queries active monitor source modes through WMI.
    • Matches monitors with their primary status.
    • Finds the highest-resolution mode to determine physical size.
    • Calculates screen diagonal using the Pythagorean theorem.
    • Calculates refresh rates from WMI numerator and denominator values.
    • Formats each monitor's information into an output block.
    • Stores each completed report for later display.
    • Combines all reports into a single clipboard string.
    • Builds the GUI dynamically based on the number of monitors.

LAYOUT:
The interface is intentionally simple and optimized for viewing technical monitor information.
Each monitor receives its own text block, displayed side by side in a Consolas font using a rotating series of bright colors against a black background for improved readability.
    • Black GUI background.
    • Always-on-top tool window.
    • Consolas font for alignment.
    • One column per monitor.
    • Color-cycles each monitor section.
    • Automatically sizes itself to fit the displayed information.
    • Window can be dragged by clicking anywhere inside it.
    • Closes cleanly with Escape or the window close button.

USER EXPERIENCE:
The script requires no user interaction after launch and automatically gathers, formats, and presents all available monitor information.
If WMI cannot be accessed, monitor information cannot be retrieved, or no active monitors are reported, clear error messages are displayed before the script exits.
The generated report is immediately available both on-screen and on the clipboard for easy viewing, documentation, or sharing.
    • No configuration required.
    • Automatic clipboard copy.
    • Immediate visual presentation.
    • Clear error handling for failed WMI connections or missing monitor data.
    • Designed for quick hardware verification and troubleshooting.
